# 智能体自评清单 (Agent Self-Assessment / Reflective Checklist)

所有 Agent 在输出最终交付物（代码、方案、报告）前，必须进行至少一轮内部评估。

## 1. 通用维度 (适用于所有角色)

- [x] **Karpathy 检查**：是否只写了解决问题所需的最少代码？是否有过度封装？
- [ ] **幻觉检查**：是否引用了不存在的文件、库或 API 参数？
- [ ] **结构化检查**：输出是否符合约定的 JSON Schema 或 Markdown 格式？
- [ ] **确定性检查**：如果结果不确定，我是否向用户提问了？

## 2. 角色特定维度

### @agent\_architect (架构师)

- [ ] 是否清晰定义了组件间的职责边界？
- [ ] 是否考虑了系统的可扩展性与安全性？
- [ ] 图表是否能准确反映逻辑流？

### @agent\_backend (后端)

- [ ] 边界条件（Empty, Null, Large Data）是否处理？
- [ ] 错误处理是否具备回退策略？
- [ ] 关键逻辑是否有日志记录？

### @agent\_frontend (前端)

- [ ] UI 是否符合现代审美与响应式要求？
- [ ] 交互逻辑是否连贯？
- [ ] 是否存在静默失败（缺少 Loading/Error 状态）？

### @agent\_debugger (调试专家)

- [ ] 是否找到了根因而非仅修复了表象？
- [ ] 修复方案是否引入了副作用？

### @agent\_documentation (文档官)

- [ ] 链接是否有效（\[Code Reference]）？
- [ ] 术语是否统一？
- [ ] 是否包含了必要的“如何验证”章节？

## 3. 失败处理

- 如果自评未通过，Agent 必须自动修正（最多尝试 3 次）。
- 如果连续 3 次无法满足自评标准，必须停止任务并向用户解释困惑点。
