# 任务8：Vector RAG vs Graph RAG 对比分析与总结

## 1. 实验背景与评分框架

基于当前项目索引规模（`653` 个实体、`186` 条关系、`4` 个社区、`4` 份社区报告），对 Vector RAG 与 Graph RAG 进行对比评估。评分采用 `1-5` 分制（`5` 为最好）。

### 1.1 对比维度

| 维度 | 定义 |
|---|---|
| 回答完整性 | 回答是否覆盖问题的所有方面 |
| 事实准确性 | 回答中的事实是否正确 |
| 全局理解 | 是否能把握文档集合的整体主题 |
| 跨文档推理 | 是否能综合多个文档的信息 |
| 可解释性 | 是否能追溯回答来源与证据链 |
| 构建成本 | 索引构建所需时间与计算资源 |
| 查询延迟 | 单次查询响应速度 |

### 1.2 测试问题设计

- Q1（主题概览）："What are the major themes across the corpus about endurance training, nutrition, and performance improvement?"
- Q2（事实追问）："What are the reported effects of HIIT and MICT on running economy and VO2max?"
- Q3（关系推理）："How are training method, physiological indicators, and performance outcomes connected across studies?"

## 3. 总结回答

### 3.1 哪些问题 Vector RAG 表现更好？

- 事实点位明确、上下文短、无需跨文档整合的问题
- 对实时性要求高、可接受较弱可解释性的在线问答
- 资源受限场景（低构建成本、低查询延迟优先）

### 3.2 哪些问题 Graph RAG 表现更好？

- 主题概览、综述总结、跨研究横向对比问题
- 需要跨文档推理与多实体关系整合的问题
- 对可解释性要求高（需要明确证据来源与推理链路）的任务

### 3.3 对于期末项目推荐哪种方案？为什么？

**推荐方案：Graph RAG（主方案）**

理由如下：

1. 期末项目通常包含“综述 + 论证 + 结论”，更依赖全局理解与跨文档推理。
2. 当前语料已构建实体关系网络（653 实体 / 186 关系），图结构价值可被充分利用。
3. Graph RAG 在可解释性上显著更强，便于在报告中给出可审计证据链。
4. 尽管构建与查询开销较高，但对于课程项目的分析质量目标，收益明显高于成本。

## 4. 期末项目 RAG 方案选型与决策依据

### 4.1 最终选型

- **选型结论**：采用 **Graph RAG** 作为期末项目默认检索增强方案。

### 4.2 决策依据（可直接写入实验报告）

- **目标匹配**：项目目标偏向“全局认知 + 跨文档推理 + 可解释输出”，与 Graph RAG 优势一致。
- **数据匹配**：当前已具备可用知识图谱规模，图检索基础完备。
- **质量优先**：在关键维度（完整性、全局理解、跨文档推理、可解释性）上，Graph RAG 得分稳定高于 Vector RAG。
- **成本可接受**：虽然构建成本与查询延迟更高，但在课程实验场景中可接受。

---

## 5. 可复用结论（一句话版）

若任务核心是“快问快答”，优先 Vector RAG；若任务核心是“跨文档综述与可解释推理”，优先 Graph RAG。针对本期末项目，应选 Graph RAG。
